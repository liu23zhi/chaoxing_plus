# Answer Work Controls And Status Colors Design

**Goal:** 为现有答题结果面板增加运行控制与单题重答能力，并用统一状态颜色让用户更直观地识别每道题的处理结果。

**Scope:** 仅扩展当前 `答题结果与学习控制` 面板，不新增独立面板；包含全局暂停/继续、当前选中题的单题重答、题号列表状态着色，以及最小的运行时状态接线。

## User-facing behavior

### 1. Global answer controls
在 `答题结果与学习控制` 顶部操作区新增运行控制组：
- 运行中显示 `暂停答题`
- 已暂停时显示 `继续答题`
- 保留现有 `切换为题目列表 / 切换为序号列表` 与 `清空结果`

运行控制只影响当前正在执行的自动答题流程，不影响已经完成的结果展示。

### 2. Retry current selected question
在当前选中题的详情区新增 `重答本题` 按钮。

交互规则：
- 用户先点击题号或题目列表项，选中某一题
- 再点击 `重答本题`
- 系统仅对这一题重新执行：搜题 → 解析 → 作答 → 刷新该题结果
- 不重新启动整轮自动答题
- 不清空其他题目的结果

这与用户选择的 B 方案一致：单题重答通过“选中题 + 独立按钮”触发，而不是在每个题号上堆按钮。

### 3. Question number colors
序号列表使用单一主状态色，避免一题多色冲突。状态优先级固定为：
1. 当前选中：蓝色
2. 已人工答题：黄色
3. 成功获取到可用答案：绿色
4. 未获取到答案 / 作答失败：红色
5. 其他默认状态：中性灰白

推荐色语义：
- 蓝色：当前查看中
- 黄色：人工介入
- 绿色：成功命中答案
- 红色：未命中或失败

### 4. Manual-answer detection
本次不做重型智能识别，而采用最小、稳妥的人工答题识别策略：
- 若某题在自动流程外被用户手动修改过答案，或检测到题目 DOM 中已有明确的人为输入/选择痕迹
- 则将该题标记为 `manual`
- `manual` 在序号列表中显示为黄色

如果当前题同时被选中，则按优先级显示为蓝色；取消选中后恢复黄色。

## State model

在现有 `workResults` 条目基础上，补充最小状态字段，建议为：
- `statusSource`: `idle | answered | unresolved | manual`
- `isSelected`: 运行时派生状态，不持久化
- `retrying`: 是否正在单题重答

其中：
- `answered` 表示成功获取并完成有效自动处理
- `unresolved` 表示未拿到答案、解析失败或作答失败
- `manual` 表示用户人工介入
- `idle` 表示尚未形成上述明确状态

也可以不新增完整枚举，而是通过现有 `finish / requested / resolved / error / searchInfos` 派生，再只增加一个最小 `manual` 标记字段。实现时优先选这个更小的方案。

## Runtime design

### 1. Pause / continue
复用现有 worker 的 `stop / continuate` 事件与 `isStop` 状态，不重写主循环。

面板层新增对“当前活跃 worker”的引用与运行状态展示：
- 自动答题启动时注册为当前活跃 worker
- 点击 `暂停答题` 时触发 worker stop
- 点击 `继续答题` 时触发 worker continuate
- worker done / close 后清理活动引用并恢复 UI 到空闲态

### 2. Retry single question
单题重答不插队进入整轮 worker 的 for-loop，而是走独立入口：
- 读取当前选中题的原始题干、题型和必要上下文
- 单独复用现有 `answerer`、`resolver`、`onResultsUpdate` 能力
- 只更新 `state.workResults.results[index]`
- 渲染面板与题目状态

这样能避免把“整轮自动答题”和“单题纠错”耦合到一起。

### 3. Required context caching
为了支持 `重答本题`，需要为每个结果条目保留最小可重试上下文。建议只缓存：
- 题干文本
- 题型
- 当前题在页面中的索引
- 必要时的题目 root / 元素定位信息（若当前作答链路必须依赖）

如果现有结果条目已经能通过索引回到原题位置，则优先用索引，不扩大持久化结构。

## Panel changes

### Numbers view
序号按钮保持当前紧凑风格，但调整颜色逻辑：
- 选中题：蓝底白字
- 成功题：绿色边框 / 绿色浅底
- 未命中题：红色边框 / 红色浅底
- 人工题：黄色边框 / 黄色浅底
- 默认题：灰边白底

`title` 继续显示状态文本，便于悬停查看。

### Questions view
题目列表项同步使用相同状态语义：
- 左侧序号或卡片边框跟随状态色
- 当前选中仍优先显示蓝色高亮
- 详情区显示 `重答本题`

### Detail area
当前题详情区新增操作行：
- `重答本题`
- 必要时可显示 `正在重答...` 的禁用态

本次不加入更多操作，保持面板克制。

## Error handling

- 没有选中题时，`重答本题` 按钮禁用
- 当前没有活跃自动答题流程时，不显示 `暂停答题`，或显示禁用态
- 单题重答失败时，仅更新当前题为红色状态，并在详情区展示失败信息
- 单题重答不应清空整面板历史结果

## Testing strategy

按最小 TDD 范围补测试：
1. 面板渲染测试
   - 顶部存在 `暂停/继续` 控制入口
   - 当前题详情存在 `重答本题` 按钮
   - 序号按钮根据状态产生对应颜色/状态标记

2. 状态派生测试
   - 成功题 → 绿色
   - 未命中/失败题 → 红色
   - 人工题 → 黄色
   - 当前选中题 → 蓝色优先

3. 运行控制测试
   - 暂停按钮接到 worker stop
   - 继续按钮接到 worker continuate
   - 单题重答只更新一个结果条目，不清空全局结果

## Files likely to change

主要修改：
- `src/projects/common.ts`

可能补充最小接线：
- `src/core/worker/interface.ts`
- `src/core/worker/worker.ts`
- 相关 `scripts/*.test.mjs`

## Non-goals

本次不包含：
- 每个题号上直接放独立按钮
- 批量重答失败题
- 自动识别所有人工操作场景的复杂 DOM 规则引擎
- 新增独立答题控制面板
