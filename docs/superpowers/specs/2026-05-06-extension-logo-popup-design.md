# chaoxing_plus 扩展图标与弹出页设计

日期：2026-05-06

## 1. 背景与目标

当前 `chaoxing_plus` 仍然是一个 Manifest V3 浏览器扩展壳层，`build.mjs` 会在构建时直接生成 `dist/manifest.json`，并复制需要的静态资源。

这次变更的目标很小且明确：

- 为扩展接入正式 logo
- 为工具栏点击增加一个轻量弹出页
- 调整扩展描述，让其更贴近“超星自动学习 / 自动搜题扩展”的用户理解

本次不修改超星业务脚本，不改变现有内容脚本注入逻辑，也不引入新的后台流程。

## 2. 已确认决策

1. 图标方案采用用户指定文件：`icon/icon_512X512.png`
2. popup 中的跳转目标固定为：`https://www.chaoxing.com/`
3. 采用方案 A：最小接入，不额外生成多尺寸图标资源
4. 改动范围只限于扩展壳层资源、popup 页面与 manifest 生成逻辑

## 3. 设计结论

### 3.1 图标接入

`build.mjs` 在生成 `manifest.json` 时增加 `icons` 字段，并将 `icon/icon_512X512.png` 复制到 `dist` 中可被 manifest 引用的位置。

由于当前只提供 512 尺寸图标，本次直接复用该文件，不新增额外图片处理流程。浏览器需要较小尺寸图标时，交由浏览器自行缩放。

### 3.2 工具栏弹出页

新增一个静态 popup 页面，作为扩展工具栏点击后的入口页。页面内容保持极简，只承担说明与跳转作用：

- 显示扩展名称
- 显示一句简短说明：这是一个给超星网站使用的自动学习 / 自动搜题扩展
- 显示一个按钮，点击后打开 `https://www.chaoxing.com/`

实现上优先采用静态 `popup.html` + 少量内联或配套脚本，不引入框架，不接入现有超星页面运行逻辑。

### 3.3 扩展描述

更新 manifest `description`，从现在偏“扩展包装层”的技术描述，改为面向用户的功能描述。

描述原则：

- 明确这是超星网站专用
- 明确其核心用途是自动学习与自动搜题辅助
- 不夸张承诺，不写与当前能力不符的内容

## 4. 组件与文件边界

### 4.1 修改文件

- `build.mjs`
  - 增加 manifest 的 `icons`
  - 增加 manifest 的 `action.default_popup`
  - 更新 `description`
  - 在构建时复制图标与 popup 静态资源

### 4.2 新增文件

- `src/popup.html` 或等价静态 popup 文件
  - 承担弹出页结构与文案
- `src/popup.js`（如需要）
  - 仅负责按钮点击打开超星官网

如果可以只用 HTML 完成目标，则不强制新增 JS；但若需要更稳定地打开新标签页，则允许加入极小的 popup 脚本。

## 5. 数据流与行为

1. 执行 `npm run build`
2. `build.mjs` 生成新的 `dist/manifest.json`
3. 构建脚本复制图标与 popup 资源到 `dist`
4. 浏览器加载扩展后：
   - 扩展管理页 / 工具栏读取 manifest 图标
   - 用户点击工具栏图标时打开 popup
   - 用户点击 popup 按钮后跳转到 `https://www.chaoxing.com/`

## 6. 错误处理与兼容策略

- 若图标文件缺失，构建应尽早暴露问题，而不是静默生成缺图标产物
- popup 不依赖内容脚本或页面注入，因此不会影响现有超星自动学习逻辑
- 本次不处理多尺寸图标适配；若后续需要更精细工具栏显示，再单独补充尺寸资源

## 7. 测试与验证

本次以最小验证为主：

1. `npm run build` 成功
2. 检查 `dist/manifest.json` 中存在：
   - 更新后的 `description`
   - `icons`
   - `action.default_popup`
3. 检查 `dist` 中存在：
   - 图标文件
   - popup 文件
4. 手动加载扩展后验证：
   - 扩展显示新 logo
   - 点击工具栏图标能打开 popup
   - 点击 popup 按钮可跳转到 `https://www.chaoxing.com/`

## 8. 范围外事项

以下内容不在本次范围内：

- 多尺寸图标自动生成
- popup 中加入复杂设置面板
- popup 与超星页面脚本联动
- 新增 background 常驻逻辑
- 修改任何学习、答题、面板或注入链路的业务行为
